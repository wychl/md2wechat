# 公众号样式

## 允许的 HTML 标签

### 块级标签

- `div`、`section` —— 常用作布局容器  
- `p` —— 段落  
- `h1` ~ `h6` —— 标题  
- `blockquote` —— 引用块（样式可能被微信重置）  
- `pre` —— 预格式化文本（注意换行和缩进可能被压缩）  
- `ul`、`ol`、`li` —— 列表（多层嵌套可能丢失样式）  
- `table`、`thead`、`tbody`、`tr`、`th`、`td` —— 表格（`colgroup`/`col` 通常被删）  
- `figure`、`figcaption` —— 图文容器（部分支持）  
- `hr` —— 水平线  
- `header`、`footer`、`main`、`article` —— 语义标签可能保留但无特殊样式

### 内联标签

- `span` —— 行内容器，最重要的样式载体  
- `a` —— 超链接（`href` 必须完整 https，订阅号外链可能受限）  
- `strong`、`b` —— 加粗  
- `em`、`i` —— 斜体  
- `u` —— 下划线  
- `s`、`del` —— 删除线  
- `code` —— 行内代码（通常需要配合 `style` 生效）  
- `br` —— 换行  
- `sub`、`sup` —— 上下标  
- `img` —— 图片（**唯一被允许的多媒体标签**）  
- `abbr`、`cite`、`q` 等语义标签可能保留但无视觉作用

### 明确被禁止的标签

`script`、`iframe`、`object`、`embed`、`video`、`audio`、`canvas`、`svg`（部分内联 SVG 可尝试但极易被过滤）、`form`、`input`、`button`、`select`、`link`、`meta`、`style`（独立标签会被整段删除，**所有样式必须内联到 `style` 属性里**）

## 属性白名单（仅这些属性可能存活）

### 全局属性

- `style` —— **唯一稳定保留的样式途径**，但属性值内部会二次过滤  
- `class`、`id` —— **会被完全移除或重命名**，不可依赖（高亮、自定义样式的基础全部丢失）  
- `data-*` —— 一般会被删除  
- 事件属性（`onclick` 等）—— 全部删除

### 特定标签属性

- `img`：`src`（必须 https）、`alt`、`width`、`height`（部分支持）、`data-src`（懒加载可能保留，但最好直接用 `src`）  
- `a`：`href`（必须完整 https 链接，`target` 可能被删，不要依赖 `target="_blank"`）  
- `td`、`th`：`colspan`、`rowspan`（**部分情况保留**，但复杂合并表格容易错乱）  
- `table`：`border`（可能被忽略）、`cellpadding`、`cellspacing`（大概率被删）  
- 其他所有 HTML 原生属性（如 `type`、`value`、`name`、`align`、`bgcolor` 等）大概率被过滤。

## 内联样式（`style` 属性）白名单

微信会二次清洗 `style` 的内容，**仅以下 CSS 属性有较高概率存活**：

### 文字与排版

- `color`、`font-size`、`font-weight`、`font-style`、`font-family`（实际仅系统默认字体生效）  
- `text-align`（left/center/right/justify）  
- `text-decoration`（underline/line-through）  
- `line-height`、`letter-spacing`、`word-spacing`  
- `text-indent`  
- `white-space`（pre-wrap 等，用于保留代码空格）

### 背景与边框

- `background-color`（`background` 简写可能被杀，最好分开写）  
- `background-image`（慎用，需 https 且易被过滤）  
- `border`（可定义宽度、样式、颜色，但圆角 `border-radius` 在部分机型失效）  
- `border-radius`（一般保留，但某些版本微信会抹除）  
- `padding`、`margin`（四个方向基本都支持，但 `margin: 0 auto` 等可能失效，因为 `width` 被影响）

### 尺寸与显示

- `width`、`max-width`、`height`（图片和容器可用，但 `%` 可能被转成 px 或删除）  
- `display`（**仅 `inline`、`block`、`inline-block` 可能保留**，`flex`、`grid`、`table-cell` 等绝对会被删）  
- `vertical-align`（有时保留）  
- `overflow`（**严重不支持**，`overflow: auto/scroll` 一定被删，无法做出滚动区域）  
- `float`、`position`、`top/left/right/bottom` —— **全部被过滤**

### 其他

- `opacity` —— 部分支持  
- `box-shadow` —— 大概率被删，或显示异常  
- `list-style` —— 不稳定，建议手动用 `•` 等符号

> 核心原则：**只用最基础的排版属性，所有样式写在 `style` 里，并做好降级测试。**

## 实战对照表（快速查阅）

| 需求 | 可用标签 | 关键样式 | 风险点 |
|------|---------|---------|--------|
| 标题 | `h1`-`h6` 或 `p`+`strong` | `font-size`、`font-weight` | `h1` 可能过大，微信有默认样式覆盖 |
| 加粗 | `strong` / `b` | 或内联 `font-weight:bold` | 两者皆可，`strong` 语义更佳 |
| 代码块 | `pre` + `code`，所有高亮必须用内联 `span style` | `background-color`、`white-space:pre-wrap`、`font-family:monospace` | **禁止用 `class` 控制高亮**，必须把高亮颜色写成内联样式 |
| 图片 | `img` | `max-width:100%`、`height:auto`（可能被删） | 必须 https，防盗链需提前上传微信素材库 |
| 表格 | `table`/`tr`/`td` | `border`、`padding`、`text-align` | 复杂合并易变形，建议列数≤4，或转为图片 |
| 引用块 | `blockquote` | `border-left`、`padding-left`、`color` | 微信可能重置左边框，样式需强加内联 |
| 列表 | `ul`/`ol`/`li` | `margin-left`、`padding-left` | 不建议依赖 `list-style-type`，直接写符号更稳 |
| 链接 | `a` | 只能通过 `color` 染色 | 外链限制、不能 `target="_blank"` |

## 避坑提醒

1. **千万不要留 `class`**：语法高亮、自定义主题的所有样式必须全部摊平到 `style` 属性里，否则一进微信样式全丢。
2. **代码块换行必须用 `pre-wrap`**：否则长代码会撑破页面。
3. **测试在微信手机端**：PC 端 Mac/Windows 的显示不能代表手机，高度依赖内联的样式在某些安卓机型上表现不一致。
4. **图片优先用微信图床**：外链图片即使加 `referrerpolicy` 也可能被拦截，建议转换流程中自动上传到公众号素材库并替换链接。
